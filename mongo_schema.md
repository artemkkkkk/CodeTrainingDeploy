# Схема базы данных MongoDB: Комментарии

Этот документ описывает структуру коллекции `comments`, которая хранит пользовательские комментарии к задачам. 
Коллекция связана с реляционной базой данных (SQL) через внешние идентификаторы (`task_id`, `user_id`).

## Коллекция: `comments`

Хранит текстовые комментарии пользователей к конкретным задачам.

### Структура документа

| Поле | Тип данных (BSON) | Обязательное | Индекс | Описание |
| :--- | :--- | :---: | :---: | :--- |
| `_id` | `ObjectId` | ✅ Да | PK (Primary Key) | Уникальный идентификатор документа в MongoDB. Генерируется автоматически. |
| `task_id` | `Int32`  | ✅ Да | ✅ | ID задачи из SQL базы данных. |
| `user_id` | `Int32`  | ✅ Да | ✅ | ID пользователя из SQL базы данных. |
| `comment` | `String` | ✅ Да | ❌ | Текст комментария. Максимальная длина зависит от лимита документа (16 МБ), но рекомендуется ограничить на уровне приложения (напр., 2000 символов). |
| `date` | `Date` | ✅ Да | ✅ | Дата и время создания комментария. Хранится в формате UTC (`ISODate`). |

### Пример документа (JSON)

```json
{
  "_id": {
    "$oid": "507f1f77bcf86cd799439011"
  },
  "task_id": 1024,
  "user_id": 55,
  "comment": "Отличная задача! Но в третьем тесте ошибка условия.",
  "date": {
    "$date": "2023-10-27T14:30:00.000Z"
  },
}
